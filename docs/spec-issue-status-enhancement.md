# 仕様書: 課題管理ステータスの拡張

## 参加者
- テストマネージャー (品質保証観点)
- 開発マネージャー (開発工数・技術観点)
- プロジェクトマネージャー (スケジュール・優先度観点)
- フォルトマネージャー (障害分析・再発防止観点)

## 1. 背景・課題

### 現状のステータスフロー
```
open → in_progress → resolved → verified → closed
                ↘      ↙         ↙
                  open (再オープン)
```

### 現状の問題点
1. **「問題なし」の解決パスがない** — 調査の結果、不具合でないと判明した場合の正式なステータスがない
2. **重複課題の処理がない** — 既に同じ課題が報告されている場合のステータスがない
3. **対応保留の仕組みがない** — 優先度やリリース計画により対応を延期する場合のステータスがない
4. **却下理由の記録がない** — なぜクローズしたのかの分類ができない

## 2. 新規ステータスの追加

| ステータス | 英語キー | 説明 | 遷移元 |
|---|---|---|---|
| 問題なし | `not_a_bug` | 調査の結果、仕様通りまたは再現不可 | `open`, `in_progress` |
| 重複 | `duplicate` | 既存の課題と同一 | `open`, `in_progress` |
| 対応保留 | `deferred` | 対応を延期（次期リリース等） | `open`, `in_progress` |

```
                            ┌──────────────┐
                            │     open     │
                            └──────┬───────┘
                                   │
                    ┌──────────────┼──────────────┐
                    ▼              ▼              ▼
              ┌───────────┐ ┌──────────┐  ┌──────────┐
              │in_progress│ │not_a_bug │  │duplicate │
              └─────┬─────┘ └──────────┘  └──────────┘
                    │
          ┌────────┼────────┐
          ▼        ▼        ▼
    ┌──────────┐ ┌────────┐ ┌────────┐
    │ resolved │ │deferred│ │not_a_bug│
    └────┬─────┘ └────────┘ └────────┘
         ▼
    ┌──────────┐
    │ verified │
    └────┬─────┘
         ▼
    ┌──────────┐
    │  closed  │
    └──────────┘
```

## 4. 完全な遷移マップ

| From | To (許可) |
|---|---|
| `open` | `in_progress`, `not_a_bug`, `duplicate`, `deferred`, `closed` |
| `in_progress` | `resolved`, `not_a_bug`, `deferred`, `open`, `closed` |
| `resolved` | `verified`, `open` |
| `verified` | `closed`, `open` |
| `closed` | `open` |
| `not_a_bug` | `open` |
| `duplicate` | `open` |
| `deferred` | `open`, `in_progress` |

## 5. 各ステータスの業務ルール

### not_a_bug（問題なし）
- **設定者:** 担当者（開発者）またはテストマネージャー
- **必須入力:** 解決メモ（理由の記載が必須）
  - 例: 「仕様通りの動作」「再現不可」「環境依存」
- **通知:** 報告者にメール通知（確認を促す）
- **報告者アクション:** 納得できなければ再オープン可能

### duplicate（重複）
- **設定者:** 担当者またはフォルトマネージャー
- **必須入力:** 関連課題の指定（重複元の課題ID）
- **通知:** 報告者にメール通知
- **報告者アクション:** 再オープン可能

### deferred（対応保留）
- **設定者:** プロジェクトマネージャーまたは開発マネージャー
- **必須入力:** 延期理由と予定時期
  - 例: 「v2.0リリース後に対応予定」
- **通知:** 報告者にメール通知
- **レビュー:** 定期的に保留中課題をレビュー（ダッシュボード表示）

## 6. UI変更

### ステータスバッジ色
| ステータス | 色 | 意味 |
|---|---|---|
| open | 青 | 新規/未着手 |
| in_progress | 黄 | 対応中 |
| resolved | 緑 | 解決済み |
| verified | 紫 | 検証完了 |
| closed | グレー | 完了 |
| **not_a_bug** | **オレンジ** | **問題なし** |
| **duplicate** | **グレー(薄)** | **重複** |
| **deferred** | **シアン** | **保留** |

### ステータス遷移ボタン
- 「問題なし」ボタン: オレンジ系
- 「重複」ボタン: グレー系
- 「対応保留」ボタン: シアン系

- 「問題なし」「重複」で閉じた課題の統計をレポート画面で確認できること

- 保留中課題の一覧表示（長期放置防止）
