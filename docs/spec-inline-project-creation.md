# 仕様書: タスク作成時のインラインプロジェクト新規作成

## 概要
タスク作成フォーム（Step 1）のプロジェクト選択ドロップダウンに「＋ 新規プロジェクト作成」オプションを追加し、別ページに遷移せずにその場でプロジェクトを作成できるようにする。

## 現状の問題
- タスク作成時にプロジェクトを新規作成するには、一度タスク作成を中断し、プロジェクトページに移動して作成後、タスク作成に戻る必要がある
- 入力中のタスク情報が失われる可能性がある

## 実装仕様

### UI変更
1. **プロジェクト選択ドロップダウン**に「＋ 新規プロジェクト作成」オプションを追加
2. 選択すると**インラインモーダル**が表示される
3. モーダルで最低限の情報（プロジェクト名、キープレフィックス）を入力して作成
4. 作成後、自動的に新しいプロジェクトが選択された状態になる

### モーダルフィールド
| フィールド | 必須 | 型 | 説明 |
|---|---|---|---|
| プロジェクト名 | * | text | プロジェクトの名前 |
| キープレフィックス | * | text | 課題キーに使用（例: WEB, EC）最大10文字、自動大文字 |
| 説明 | | textarea | プロジェクトの説明 |
| クライアント | | select | 既存クライアントから選択 |

### 動作フロー
1. ユーザーがプロジェクト選択で「＋ 新規プロジェクト作成」を選択
2. モーダルが開く（タスクフォームの入力値はそのまま保持）
3. プロジェクト情報を入力して「作成」ボタンを押す
4. Supabaseにプロジェクトが作成される
5. プロジェクト一覧が再取得される
6. 新しいプロジェクトが自動選択される
7. クライアントが設定されている場合、タスクのクライアントも自動入力される

### 影響ファイル
- `src/components/tasks/TaskForm.tsx` — プロジェクト選択UIにモーダルトリガー追加
- `src/lib/i18n/translations.ts` — 翻訳キー追加
